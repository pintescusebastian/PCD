# PCD Homework 1 — Network Protocol Benchmark

Performance benchmark for TCP, UDP and QUIC protocols, measuring transfer time, throughput and data integrity for large data transfers (500 MB and 1 GB).

## Requirements

- Linux / WSL2 (Ubuntu 24.04)
- GCC 13+
- msquic 2.5.6
- OpenSSL (for QUIC TLS certificate)

```bash
sudo apt install libmsquic openssl -y
sudo ln -s /usr/lib/x86_64-linux-gnu/libmsquic.so.2 /usr/lib/x86_64-linux-gnu/libmsquic.so
```

## Build

```bash
make
```

Binaries are placed in `build/`.

## Usage

Start a server, then run the client in a second terminal.

```bash
# TCP
./build/server_tcp
./build/client_tcp  -s <block_bytes> -d <500m|1g> -m <stream|saw>

# UDP
./build/server_udp
./build/client_udp  -s <block_bytes> -d <500m|1g> -m <stream|saw>

# QUIC
./build/server_quic
./build/client_quic -s <block_bytes> -d <500m|1g> -m <stream|saw>
```

| Option | Description |
|--------|-------------|
| `-s`   | Block size in bytes (1 – 65535) |
| `-d`   | Data amount: `500m` or `1g` |
| `-m`   | Transfer mode: `stream` or `saw` |

## Example

```bash
./build/server_tcp
./build/client_tcp -s 1000 -d 500m -m stream
```

## Features

- **3 protocols** — TCP (port 9001), UDP (port 9002), QUIC/msquic (port 9003)
- **5 block sizes** — 1, 500, 1 000, 10 000, 65 535 bytes
- **2 transfer modes** — Streaming and Stop-and-Wait
- **SHA-256 integrity check** — client and server hashes are compared at end of session
- **QUIC** — uses msquic 2.5.6 with TLS 1.3 (auto-generated self-signed certificate)
